ABF Unternehmensverbund

Relaunch Masterplan Q2 2025

Executive Summary

Dieser Masterplan beschreibt den umfassenden Relaunch der ABF Unternehmensverbund Website für Q2 2025. Basierend auf einer detaillierten Analyse der aktuellen Website und den Anforderungen an eine moderne, leistungsfähige Online-Präsenz.

Technologie

Next.js 14+ mit React, TypeScript und Sanity CMS für moderne, performante Web-Experience

Branding

Einheitliche Corporate Identity über alle Unternehmensbereiche mit konsistentem Design-System

Zielgruppen

Multi-Audience-Ansatz für Patienten, Fachkreise, Recruiting und Investoren/Partner

Timeline

Q2 2025 Launch (März-Juni), 17 Wochen agile Entwicklung mit iterativem Ansatz

3.2 User Journeys

Persona 1: Patient mit chronischer Erkrankung

  1. Einstieg: Startseite → "Ich bin Patient"
  2. Orientierung: Therapiebereich finden (z.B. Onkologie)
  3. Information: Services verstehen (Bestellung, Beratung)
  4. Aktion: Medikament bestellen oder Kontakt aufnehmen

Persona 2: Arzt/Praxis

  1. Einstieg: Startseite → "Ich bin Arzt/Apotheker"
  2. Ziel: Fachportal erreichen
  3. Aktion: Login ins Bestellsystem
  4. Discovery: Fortbildungsangebote entdecken

Persona 3: Job-Suchende/r

  1. Einstieg: Startseite → "Ich suche einen Job"
  2. Exploration: Karrierebereich durchsuchen
  3. Evaluation: Benefits & Kultur kennenlernen
  4. Aktion: Bewerbung absenden

4. Technische Architektur

4.1 Tech Stack

Frontend

  • Framework: Next.js 14+
  • Language: TypeScript
  • Styling: Tailwind CSS + Custom Design Tokens
  • Components: React + Radix UI

Backend & CMS

  • CMS: Sanity.io
  • API: Next.js API Routes
  • Auth: NextAuth.js (für Portale)
  • Forms: React Hook Form

Infrastructure

  • Hosting: Vercel (Edge Network)
  • CDN: Vercel Edge + Sanity CDN
  • Analytics: Plausible oder Matomo
  • Monitoring: Sentry

DevOps

  • Version Control: Git + GitHub
  • CI/CD: GitHub Actions
  • Testing: Jest + Playwright
  • Code Quality: ESLint + Prettier

4.4 SEO-Optimierung

Technische SEO-Maßnahmen:
  • Meta-Tags: Dynamische Title & Description pro Seite
  • Open Graph: Social Media Previews für alle Seiten
  • Schema.org: Organization, LocalBusiness, BreadcrumbList, Service, Product
  • Sitemap: Automatisch generiert via Next.js
  • Robots.txt: Optimiert für Crawler
  • Canonical URLs: Duplicate Content Prevention
  • Alt-Texte: Pflichtfeld im CMS für alle Bilder
  • Semantisches HTML: Korrekte Heading-Hierarchie

10. Design-Prinzipien

10.1 Core Design Principles

1. Vertrauen

Professionell, medizinisch präzise, wissenschaftlich fundiert. Keine übertriebenen Marketing-Claims, sondern Fakten und Kompetenz.

2. Menschlichkeit

Warme Bildsprache, empathische Texte, echte Menschen statt Stock-Photos. Patienten stehen im Mittelpunkt.

3. Klarheit

Einfache Navigation, klare Hierarchien, strukturierte Information. Nutzer finden schnell, was sie suchen.

4. Zugänglichkeit

Barrierefrei (WCAG 2.1 AA), mobile-first, leichte Sprache-Option. Für alle Menschen nutzbar.

5. Konsistenz

Einheitliches Design-System über alle Bereiche. Wiedererkennung und Vertrautheit.

6. Performance

Schnelle Ladezeiten, optimierte Bilder, effizientes Caching. Zeit ist wertvoll.

10.2 UI-Pattern-Bibliothek

Hero-Sections

  • Full-Width Hero: Startseite, Division-Seiten
  • Split-Screen Hero: Text + Bild (50/50), Service-Seiten
  • Minimal Hero: Nur Headline + Breadcrumb, Content-Seiten

Card-Layouts

  • Service Cards: Icon + Headline + Text + CTA
  • News Cards: Bild + Date + Headline + Excerpt
  • Therapy Cards: Kategorie + Titel + Services + Link
  • Team Cards: Foto + Name + Position + Bio

Navigation-Pattern

  • Mega-Menu (Desktop): 2-Level-Navigation mit Icons
  • Hamburger-Menu (Mobile): Slide-in mit Accordion
  • Audience-Selector: 4 prominent platzierte Buttons auf Startseite
  • Breadcrumbs: Auf allen Unterseiten
