2. Über ABF Unternehmensverbund

Unternehmensprofil

Der ABF Unternehmensverbund ist ein familiengeführtes pharmazeutisches Unternehmen mit über 45 Jahren Erfahrung in der Versorgung von Patienten. Wir verstehen uns als pharmazeutischer Rundumversorger mit Fokus auf spezialisierte Therapiebereiche.

Kernwerte

  • Vertrauen verpflichtet – Familiäre Unternehmensführung seit über 45 Jahren
  • Patientenzentrierung – Individualisierte pharmazeutische Versorgung
  • Regionale Verwurzelung – Standort in der Metropolregion Nürnberg
  • Nachhaltigkeit – Vier Säulen: Ökologie, Soziales, Ökonomie, Governance

Unternehmensbereiche

ABF-Apotheke

Retail-Apotheke mit digitalen Services, Medikamentenbestellung und spezialisierte Beratung für Patienten

ABF-Pharmazie

GMP-konforme Herstellung, Lohnproduktion, Zytostatika-Herstellung und individuelle pharmazeutische Lösungen

ABF-Synergie

Shared Services und administrative Dienstleistungen für alle Unternehmensbereiche

ABF-Campus

Fort- und Weiterbildung für Ärzte und medizinisches Fachpersonal (Onkologie, Ophthalmologie, etc.)

Zielgruppen

  1. Patienten (B2C): Chronisch kranke Patienten mit spezialisiertem Therapiebedarf (Onkologie, Hämatologie, MS, etc.)
  2. Fachkreise (B2B): Ärzte, Apotheker und medizinisches Personal mit Bedarf an Fachinformationen, Bestellportalen und Fortbildungen
  3. Bewerber: Potenzielle Mitarbeiter, insbesondere PTAs, Pharmazeuten und medizinisches Fachpersonal
  4. Geschäftspartner/Investoren: Business Development, Unternehmenskommunikation, Nachhaltigkeitsreporting

Aktuelle Website

URL: http://a-b-f.de

Die aktuelle Website ist funktional, aber nicht mehr zeitgemäß. Hauptprobleme:

  • Veraltetes Design und Technologie
  • Suboptimale Performance (hohe JavaScript-Last)
  • Fehlende SEO-Optimierung (keine Alt-Texte, Social-Media-Tags)
  • Nicht optimal für Multi-Audience-Ansprache
  • Mobile-Optimierung verbesserungswürdig

3.2 Technische Anforderungen

Must-Have (verpflichtend):

  • Frontend: Next.js 14+ (App Router), React 18+, TypeScript
  • Styling: Tailwind CSS oder CSS-in-JS (styled-components, Emotion)
  • CMS: Headless CMS (Sanity bevorzugt, alternativ: Contentful, Strapi)
  • Hosting: Vercel, Netlify oder vergleichbare Edge-Plattform
  • Performance: Lighthouse Score 95+, LCP < 2.5s, CLS < 0.1
  • SEO: Server-Side Rendering, dynamische Sitemaps, Schema.org Markup
  • Accessibility: WCAG 2.1 Level AA konform
  • Responsive: Mobile-First, optimiert für alle Geräte

Nice-to-Have (optional):

  • Storybook für Komponenten-Dokumentation
  • E2E-Testing mit Playwright oder Cypress
  • CI/CD mit GitHub Actions
  • Monorepo-Setup (Turborepo/NX) für skalierbare Architektur

5. Detaillierte technische Spezifikationen

5.1 Frontend-Requirements

Framework & Libraries:

  • Next.js 14+ mit App Router (nicht Pages Router)
  • React 18+ mit Server Components
  • TypeScript (strict mode)
  • Styling: Tailwind CSS (bevorzugt) oder CSS-in-JS
  • State Management: React Context, Zustand oder Redux (falls nötig)
  • Forms: React Hook Form + Zod/Yup Validation

Performance-Anforderungen:

Metrik Zielwert Messung
Lighthouse Score 95+ PageSpeed Insights (Desktop & Mobile)
LCP (Largest Contentful Paint) < 2.5s Core Web Vitals
FID (First Input Delay) < 100ms Core Web Vitals
CLS (Cumulative Layout Shift) < 0.1 Core Web Vitals
TTFB (Time to First Byte) < 600ms WebPageTest
Bundle Size (Initial) < 200 KB Webpack Bundle Analyzer

5.2 CMS-Requirements

Präferierte Lösung: Sanity.io

  • Strukturiertes Content-Modeling (Schemas für alle Content-Types)
  • GROQ-Queries für flexible Content-Abfrage
  • Preview-Funktion (Draft-Mode in Next.js)
  • Asset-Management (Bilder, PDFs, Videos)
  • Versionierung & Content-Historie
  • Mehrsprachigkeit-Vorbereitung (auch wenn initial nur DE)
  • Custom Workflows (Optional: Approval-Prozesse)

Alternative CMS (falls begründet):

  • Contentful
  • Strapi (selbst gehostet)
  • Payload CMS
Wichtig: Das CMS muss redakteurfreundlich sein. Technische Kenntnisse des Content-Teams sind begrenzt. Intuitives UI ist entscheidend.

5.3 SEO & Accessibility

SEO-Features (verpflichtend):

  • Server-Side Rendering (SSR) oder Static Site Generation (SSG) wo möglich
  • Dynamische Meta-Tags (Title, Description) pro Seite via CMS
  • Open Graph & Twitter Cards für alle Seiten
  • Schema.org Markup:
    • Organization
    • LocalBusiness
    • BreadcrumbList
    • Service/Product (für Therapiebereiche)
    • JobPosting (für Karriere-Seiten)
  • XML-Sitemap (automatisch generiert)
  • Robots.txt
  • Canonical URLs
  • Alt-Texte (Pflichtfeld im CMS)
  • Semantisches HTML (korrekte Heading-Hierarchie)

Accessibility-Features (WCAG 2.1 AA):

  • Keyboard-Navigation für alle Interaktionen
  • Screen-Reader-Optimierung (ARIA-Labels wo nötig)
  • Farbkontraste min. 4.5:1 (Text) und 3:1 (UI-Elemente)
  • Touch-Targets min. 44x44px
  • Focus-Indikatoren für alle interaktiven Elemente
  • Skip-Links für Navigation
  • Responsive Text-Sizing (kein fixed px)
  • Accessibility-Testing mit axe DevTools oder WAVE

5.4 Hosting & Infrastructure

Anforderungen:

  • Edge-Network: CDN für schnelle weltweite Auslieferung
  • SSL/TLS: HTTPS (A+ Rating bei SSL Labs)
  • Uptime: 99.9% SLA
  • Backups: Automatisch, täglich
  • Monitoring: Uptime-Monitoring, Error-Tracking (z.B. Sentry)
  • Environments: Production, Staging, Preview
  • CI/CD: Automatisches Deployment bei Git-Push

Bevorzugte Plattformen:

  • Vercel (bevorzugt für Next.js)
  • Netlify
  • Cloudflare Pages

5.5 Security & Compliance

  • DSGVO-Compliance: Cookie-Banner, Datenschutz-Seite, Impressum
  • Security Headers: CSP, X-Frame-Options, etc.
  • Form-Security: CSRF-Protection, Rate-Limiting, Captcha (bei Bedarf)
  • Dependency-Scanning: Automatische Checks auf vulnerable Packages
  • Code-Signing: Nur signierte Commits (optional)

6. Projektmanagement & Kommunikation

6.1 Projektmethodik

Wir bevorzugen agile Projektmethoden mit iterativen Sprints und regelmäßigen Reviews.

Erwartete Methodik:

  • Sprints: 2-Wochen-Sprints mit Sprint Planning, Daily Standups (optional), Sprint Review, Retrospektive
  • Kommunikation: Slack/Teams für tägliche Kommunikation, wöchentliche Status-Calls
  • Tools: Jira, Asana, Linear oder vergleichbar für Task-Management
  • Versionskontrolle: Git (GitHub, GitLab oder Bitbucket)
  • Design-Collaboration: Figma (bevorzugt) oder Adobe XD

6.2 Stakeholder & Entscheidungswege

ABF-seitige Ansprechpartner:

  • Projekt-Owner: Geschäftsführung (finale Freigaben)
  • Projekt-Manager: Interne Koordination (täglicher Kontakt)
  • Fachliche Ansprechpartner: Vertreter aus allen 4 Divisions
  • Content-Team: 2-3 Personen für Content-Erstellung und -Pflege

Freigabeprozesse:

  • Design-Freigaben: Wireframes → Design-System → Designs (pro Template-Typ)
  • Content-Freigaben: Pro Bereich (Über ABF, Divisions, etc.)
  • Development-Freigaben: Pro Sprint (funktionale Increments)
  • Go-Live-Freigabe: Nach finaler QA und Abnahme
